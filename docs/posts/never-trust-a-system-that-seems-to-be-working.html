Never trust a system that seems to be working

so a fun thing about f*ctorio's space exploration mod is that you can screw up and inadvertently recreate a pivotal moment from Robert Heinlein's The Moon Is A Harsh Mistress: the bombing of earth using the lunar grain deliveries!

you have interplanetary delivery cannons (mass drivers, basically) that let you launch payloads to other planets, but an important thing is that you have space at the destination to accept them.

if you don't, instead of the cargo coming in for a nice soft landing in a delivery-cannon-chest, it doesn't slow down and hits the surface at orbital velocities. this tends to make your storage problem worse, as your warehouse is now heavily damaged.

and the next shipment is already on the way, so... pretty soon you get a message saying like "planet FOOBAR is missing some materials it needs for rebuilding: * delivery cannon chest * transmitter next to it * warehouse * half your fucking base "

because instead of a base you now have a bunch of cargo strewn across the landscape where the base used to be

and there's a way to avoid this: signal transmitters/receivers. these send circuit messages between planets, so your delivery cannon can know if there is space on the destination, and not launch if not

HOWEVER there is a very easy mistake to make. very easy. you will do this simply and it will fucking kill you

and that's this. delivery chest, storage unit of some kind, and you circuit up the storage to a transmitter. now the storage is broadcasting its inventory.

and then on the sending end, you have this. a signal receiver, and a delivery cannon. and you circuit up the inserter...

the inserter is set to only insert a delivery cannon capsule if the destination storage has under 8000 ice in it.

this is the worst way to do it, because it is a trap. it will appear to work, and then it will blow up your base.

and this is why. WHAT HAPPENS WHEN YOUR RECEIVING BASE LOSES POWER?

maybe you have unexpected power drain because of too much manufacturing going on at once and your accumulators run out of juice. maybe you run low on coal or uranium and start to brown out

in any case, losing power means the transmitter stops transmitting. and here's the fun part: your circuit which controls that inserter is set to insert "if [ICE] < 8000"

and GETTING NO SIGNAL AT ALL counts the same to it as ICE=0. 0 is < 8000.

so if your destination transmitter stops transmitting for any reason, the launching station will think it is low on materials and KEEP LAUNCHING. until your warehouse fills up and starts exploding and then your base explodes and now there's ice everywhere

So, here's how you fix this interplanetary logistics fuckup: You make the destination smarter, and the active part of the logic relationship

you put a decider combinator in there, which does that same test you were doing, like [ICE] < 8000, but then when that condition is true, it actively sends a signal. You can use letters or whatever, but it's easiest to just make it the item it was requesting.

and then on the launch end, you just change the inserter condition so that it's "if [SIGNAL] > 0"

so why does this make a difference? because of what happens when the transmitter fails.

now the logic is "transmit this signal ([ICE]) if you need cargo, and don't if you don't"

meaning that if the cargo-receiving end fails for any reason, it sends nothing, and the launching end... launches nothing.

instead of launching on failure, which is completely unsafe, it fails-safe. it fails to "launch nothing" which is a much better problem to have!

"whoops how come the base on Planet X has no ice?" is a low priority problem that you can fix at your leisure. "hey how come the base on Planet X FUCKING EXPLODED?!" is a high-priority problem that you better fix right the fuck now

anyway this story comes to you after I carefully rebuilt my whole interplanetary logistics infrastructure to be smart about this but I missed one, singular, delivery cannon, launching ice to a waterless planet.

and I woke up the next day to "hey planet X needs a lot of repair robots and is missing all these items" and I check the interplanetary viewer thing and OH BOY THAT'S A BIG FIELD OF ICE WHERE MY BASE USED TO BE

planet X is not running out of ice anytime soon. especially because it's not using it very fast anymore, now that the base exploded.

and this is a problem that I have run into so many times in computers. never trust a system that seems to be working.

the newbie says "aww, why isn't it working?" the intermediate says "yay, it's working!" the expert goes "hmm, why is it working?"

a working system will ALL TOO OFTEN not be working the way you think it's working.

and you think you know the parameters of how it's working. you know what could break it, how much overhead it has, what side effects it has.
