This is a commonly held false belief. In mathematics -1 and 9 are two different ways to write the same element of Z modulo 10. They mean the exact same thing. There's no issue with saying 3^2 = -1 (mod 10), for example.

Quotient/remainder are customarily defined so that if q=a/b and r=a%b, q*b+r = a. The unfortunate negative mod in C (used to be impl-specific, now required) is the remainder that goes with the now-mandated truncating division.

If you want a nicer mod, you need to either change the quotient to make the remainder non-negative (->Euclidean division), use a mod that does not correspond to the remainder for the choice of quotient, or have separate remainder and mod operations.
