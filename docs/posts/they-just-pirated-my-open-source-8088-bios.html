I know next to nothing about APL, but I have boggled at the number of APL-specific symbols in the "Miscellaneous Technical" Unicode block. And presumably it used plenty more non–APL-specific symbols. Could anybody remember them all?

Most of the symbols in APL are upper and lower case geek letters, plus a few extras like arrows, common relational symbols and quad (box). Some were originally created by overstriking (combining) two characters, like domino (box + division), so it actually isn’t too difficult to remember. And symbols often have two or more context dependent functions, so it doesn’t need as many as you might think

I get that today's hardware is so much more powerful. But my first computer was an Apple //e and I fell in love with the 6502. I quickly progressed to writing all my stuff in assembly. The instruction set was simple and quickly memorized, along with the CPU cycles for each instruction. You would count cycles in your inner loop to optimize performance. I loved that the system was simple enough that you could know everything that was going on down to the bare metal. I think that's why today I enjoy doing projects with microcontrollers because there's not much of a software stack between you and the hardware. Today it seems like so much of programming is trying to learn how to use some library and figure out why it doesn't seem to be doing what you think it should be doing. If I can't figure something out, I'd rather it be my own code!
