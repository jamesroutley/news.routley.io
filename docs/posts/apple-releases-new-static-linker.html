Apple Releases New Static Linker

Today at WWDC we introduced a new static linker. It is a ground-up rewrite that’s up to 5x faster than ld64. The new linker is written with multicore in mind, and it’s the first production ready parallel linker officially supported for iOS development. (1/n)

When I started working on linkers about 10 years ago, linking was (mostly) a serial operation. Many had dismissed the idea of a parallel linker.

Throughout the years, there has been a gradual reimagining of linkers to embrace parallelism. ld64 started performing input files reading and parsing, as well as few of the operations in the linker backend, such as writing final sections to the output file — in parallel.

When designing the new linker, we decided to go all-in. Every single operation is now performed in a parallel fashion. From symbol resolution to building linker metadata structures (debug notes, fixups, export trie etc.).

As a result, the new linker is able to link very large binaries in a fraction of the time, making the whole edit-build-debug cycle much faster and interactive for developers.
