Mastodon: Add support for incoming rich text

This is not about choosing between Markdown or HTML. ActivityPub and Mastodon already use HTML for sending messages, and this is about rendering more HTML tags correctly.

Currently if someone tries to shovel Markdown-like content it comes out like this:

<p>Hello *World*</p>
<p>* Bullet item 1</p>
<p>* Bullet item 2</p>

This causes the issue with accessibility as readers read out loud those * and other Markdown-like content.

That obviously makes no sense, this PR allows to render of more tags, and further down the line Markdown support can be added with a source field like @cam-rod just mentioned.
