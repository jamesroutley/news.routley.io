Coding Together with Jupyter

Recently I’ve been using Jupyter notebooks for a lot of my coding. For those that aren’t familiar with it, Jupyter is a programming environment for Python (and it supports many other languages) that’s very popular in the data science and machine learning communities. One of the main appeals is the ability to write and run scripts interactively. This shortens the feedback loop and allows for very quick iteration, making it easy to inspect the data and the intermediate state of the program being worked on, which in turn helps in writing correct programs. It looks something like this:

Today I found out about a feature that makes brings all of the goodies of Jupyter into the world of pair programming: collaborative editing. Since the 3.1 release in mid 2021 multiple users can now seamlessly work on Jupyter notebooks together. It does by using CRDTs via the Y.js library.

This means that you can now spin up Jupyter on a local computer or a VPS and expose it to the internet using something like Cloudflare Tunnel to easily have a place to pair in real-time in just a few minutes. You can see what it looks like below.

The best part is that no complicated setup is required. All you have to do is run Jupyter the same exact way as always, and add the extra flag `—collaborative`. That’s it!

I cannot overstate how cool this is. Especially with the normalization of remote work, having the ability to work and debug code together in a seamless way is incredibly powerful. I’m gonna start using this for my paring sessions here at RC, as well as for hosting programming interviews in the future. Hope you find this as useful as I have.
