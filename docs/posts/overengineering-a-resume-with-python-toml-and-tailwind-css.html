Overengineering a Resume with Python, TOML, and Tailwind CSS

2024-09-05 • 2 minutes to read

A “onesize fits all” resume doesn’t work for me. I’ve been programming for over a decade, and my experience is quite varied. Shipping one resume to everyone means there will often be useless details presented in a suboptimal ordering. Wasting an ordinary hiring manager’s time is bad enough, but I want to work at a startup, and since I’ve been a founder before, I know how much founders will appreciate a resume stripped down to only show relevant experience that is presented in a reasonable order.

For example, my last real job was at Heap, a product analytics company that became a unicorn at the end of 2021. A resume entry for this position could look like this:

Heap

  • Optimized SQL query performance to run across Heap’s distributed Postgres cluster with a p90 of < 10 seconds
  • Full stack web development of digital product analytics application (React, Typescript, Express)
  • Started technical writing group that produced 6+ articles that made it to the front page of HN and generated over 70k views for the company website

But what if the company doens’t care about distributed systems experience? In that case, the first bullet is irrelevant. Wasted time! What if they highly value leadership experience? Probably should have led with the third bullet point then.

Last time I was on the hunt, I magaged this sort of thing manually with a google doc. Not this time. Now, I’ve got a python cli that reads a toml file with tagged experiences that are filtered, sorted, and styled specifically for each position I apply to. The resume is rendered with an HTML Jinja template and Tailwind css.

This is over-engineered. There’s probably a clever way to do this in LaTeX, but at least I didn’t use React ;). Here I want to take you through some of the more fun parts of the code.
