Putting in the Hard Work

personal

"If you work hard and study hard. And you fuck up. That's okay. If you fuck up and you fuck up, then you're a fuckup."

Last month, I built a little daemon to plug into a local LLM model to help with my poor typing via dictation. I’ve since abandoned the tool. Why? A few things, but primarily the model was great for capturing the gist of what I was trying to say, but failed to really speed things up. I tracked it! On average, it took the same amount of time to just grind out my thoughts typing away than to dictate and edit it. The model would often capture things inaccurately or change words, mistake a word I said for a different word, or drop a whole sentence or more (I dunno… model is lazy too?) altogether. All these things would need to be proof-read and fixed up. It got worse the longer a segment or set of thoughts was. There’s still usefulness to this soft of tool, no doubt, but it turned out not to be the best solution for my own needs.

Sooo now a month later, I’m taking a different approach and starting to do the hard work to get better at typing. I am using https://www.keybr.com/ to learn and get good the Workman keyboard layout.

A/N: Please don’t hurt me Colmak fans! 😰 I mostly liked that Z/X/C/V keys were still (roughly) in the same spot as QWERTY.

How will that help? I figure that by using a different layout I can break my dependency on the keyboard labels, because they will no longer point the right keys at all, forcing me to touch type. It’s going alright so far, I’m up to around 18wpm with doing a minimum of 30 minutes practice on it each day for the last 2 weeks or so. The goal is to get to about 40wpm (my best with QWERTY is only ~35wpm) and then switch out the OS layout for Workman.

This has all reminded me that:

  1. Doing the hard work is important and it’s how we grow!
  2. I need a split columnar keyboard (because why the heck are we still typing on this crazy offset alignment layout from the typewriter era?! RIP my pinkies.)
