IMbackK

Jan 21, 2025

The technical reason for this is:

  1. some chips of one generation have different instructions (like gfx803 and gfx802)
  2. some chips of one generation have different numbers of registers (like gfx11 variants)
  3. rocm shares llvm targets with games, for games it is useful to compile a shader per GPU chip, since the compiler can do some extra optimizations knowing cache sizes and cu counts and game shaders are jit anyhow so the nummber of llvm targets dosent matter.

1,2 are actual isa differences that mean that code compiled for one really wont work on the other gpu

Over all rocm for technical reasons (like the limit of shared objects to 2GB) cant realistically support all chips in the official binaries (but can and mostly dose in source) until amd finishes work on gfxX-generic targets and the amdgcn spriv ir flavor.
