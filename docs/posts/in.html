things that i consider to be anti-antihuman.

software

  • no lock in. users should not be only able to put data into software, but take their data out. difficulty of getting data out should not be 10x of the difficulty of putting data in.
  • data vs ui. do not focus on perfect UI. focus on the ability to view data in a lot of different ways.
  • no tracking for ads. if there is a need to collect analytical data - try to expose it back to the user.
  • people-first. strive to write software that empowers people and not plays on people's weaknesses.

communication

  • speak to people same way how you speak in real life. this is the only way to get as close to "authenticity" as possible.
  • find your voice. it's hard enough to find your own writing voice. when you found it - use it. do not use someone else's voice. especially the corporate

support

  • talk in your own voice. see points on communication above.
  • if you feel something and want to say something - say it. if no - totally ok.
