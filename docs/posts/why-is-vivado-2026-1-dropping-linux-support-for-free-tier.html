        Hello @filisavi (Member)​ ,

        Vivado ML Standard Edition v2025.2 is going to be officially supported (i mean if there are any bugs found, these can be fixed) until v2026.3 release, as any release older than the current 3 released versions of Vivado, is then becomes unsupported (meaning no bugs will be fixed with Vivado Standard Edition v2025.2 after Vivado v2026.3). However, users can continue using V2025.2 forever, if they wish to do so.

        Also, Vivado ML Standard Edition v2025.2 is license free, meaning no physical Webpack license is needed. Vivado won't be looking for any Vivado related license file. Users only need to obtain and use any IP Core related licenses, or Vivado Model Composer (for SysGen).

        Also, as you can see, all our previous versions of Vivado are still there, in the Vivado Archive folder (under the Downloads page), even our very first versions of Vivado, released in 2012.

         

        With new Tier based licensing scheme, if you must move on and use Vivado v2026.x, and also need Linux support, users should move to the Core based tier licensing.

        For student, we now give free 60-days full Enterprise related license, meaning that there is 60-day of free licensing, that should be enough to complete their designs that they are building for learning purposes (targeting larger devices that are not supported by BASIC tier license level).

        If the device they are designing with is a BASIC supported device, the users need a Windows OS in this case.

        Almost all our surveys show that it is close to 70% of the customers are still using Windows.

        And AMD is not planning dropping the full support of Windows OS, for AMD tools such as Vivado!
