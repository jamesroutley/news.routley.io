Summarize Any Article as a Dialog Between Two People with Bing Chat

I recently discovered Recast, a brilliant tool that turns your want-to-read articles into rich audio summaries. It summarises long articles into the human dialog.

Here is an example of Recast audio file:

Because I was fascinated by this idea I tried to replicate it with the help of ChatGPT Bing Chat. My idea is simple, use Bing Chat to create a dialog summary and later use Microsoft Edge Browser for text-to-speech conversion.

Here is the prompt for Bing Chat:

browse https://electrek.co/2023/05/15/renewables-us-utility-scale-generating-capacity-q1-2023/ and make a summary in the form of a dialog between two people. Leave an empty row after each dialog. Bolden the important parts of the text. Before the dialog start with 2 or 3 sentence summary.

Change the URL with your URL.

So you have your summary in dialog form.

It also works with PDFs.

browse https://arxiv.org/pdf/2305.07185.pdf and make a summary in a form of dialog between two people. Leave an empty row after each dialog. Bolden the important parts of the text. Before the dialog start with 2 or 3 sentence summary.

And Bing Chat response:

The paper proposes Megabyte, a multi-scale decoder architecture that enables end-to-end differentiable modeling of sequences of over one million bytes. Megabyte segments sequences into patches and uses a local submodel within patches and a global model between patches. The paper shows that Megabyte can perform competitively with subword models on long context language modeling, achieve state-of-the-art density estimation on ImageNet, and model audio from raw files.

A possible dialog based on the paper is:

**Alice**: Hey Bob, have you read the new paper on Megabyte?

**Bob**: No, what's that?

**Alice**: It's a new model for predicting million-byte sequences with multiscale transformers.

**Bob**: Million-byte sequences? That sounds challenging.

**Alice**: Yeah, it is. Most existing autoregressive transformers can only handle short sequences like text or low-resolution images. They scale poorly to long sequences like high-resolution images, podcasts, code, or books.

**Bob**: So how does Megabyte solve this problem?

**Alice**: Well, it uses a clever trick. It segments the sequences into patches and uses a local submodel within patches and a global model between patches.

**Bob**: What does that mean?

**Alice**: It means that it can use sub-quadratic self-attention, much larger feedforward layers, and improved parallelism during decoding. This makes it faster and more efficient than standard transformers.

**Bob**: I see. And does it work well?

**Alice**: Yes, it does. The paper shows that Megabyte can perform competitively with subword models on long context language modeling, achieve state-of-the-art density estimation on ImageNet, and model audio from raw files.

**Bob**: Wow, that's impressive. I wonder what kind of applications it can enable.

**Alice**: Me too. Maybe we can use it to generate high-quality content without tokenization.

**Bob**: That would be awesome. I'll check out the paper. Thanks for telling me about it.

But why not make it one step further and make Microsoft Edge read the text?

The problem with Microsoft Edge is that text-to-speech always starts from the top of the webpage. But we only what to read the main text generated with Bing Chat. I remembered that back in the day I made the Writability – Minimal Distractions Free Browser Text Writing Tool that can help us copy the text in the browser and force the Edge to read it.

You need to paste this code inside the Microsft Edge URL

data:text/html;charset=utf-8, <title>Writability</title><body OnLoad='document.body.focus();'  contenteditable style="font-size:21px;line-height:1.6;font-family:'Chaparral Pro',Georgia;max-width:21em;margin:0 auto;padding:3rem;background-color:rgb(233,233,225);color:rgb(68,68,68);" spellcheck="false">

And paste any text for Edge text-to-speech to read it.
