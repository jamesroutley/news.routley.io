  • girl (non-gender-denominational) what did i even do in week 26??
  • i got to 58 days of daily journalling - today would've been day 60 - but i couldn't be bothered yesterday on some deep existential level so i missed a day. luckily, given your definition of luck, 750 words has a "streak fairy" feature where if I write 1500 today i can keep my streak intact. but like, i still know it's broken so what then?
  • i've been working on snapback, which is my actual name for the system (app? website? both? something else?) i'm trying to make to track my habits. it was codename "no good cookie" for a while, for reasons i've forgotten. i wrote it in svelte and dislike how it is turning out so far so i'm gonna start again.
  • on snapback, a friend is helping me think about the "data model design" which sounds esoteric to me but has been explained as "what the essential data structures and relationships would be whether it was run from the command line or an android app".
  • the whole thing with the data modelling has made me think "why don't more people write about how they approach side projects" and I think I've answered for myself that: a) they like to show a finished project and step back through it, which will always be less raw but also less embarrassing if you decide you no longer care about the project. it's also way easier to come up with a coherent narrative. b) this one starts off with a tangent. as much as i want to be a scrappy fiddle person I think I don't think in discreet parts like that (yet?) so i would be showing off things that are broken in some way (beyond very simple projects). i once met someone at a meetup who told me he had been working on a side project for years but didn't feel it was good enough to release. I suggested maybe releasing it and getting some feedback and he was very upset about the concept. Long side to say that I get there's a bunch of points between embryonic, broken and "perfect in an infinite amount of time and you will never see it before then". it's scary to share a beautiful complete project that might flop (even in the sense of no likes, comments, emails or views), or have the equally scary but smaller possibility of your sketched out project being seen as irredeemably boring/contrived/pretentious/awful before it has a chance to unfurl fully, but yeah. c-z) the same stuff that stops people blogging and/or coding in the first place stops them making tiny scrappy things and talking about them.
  • all this to say: i'm torn on whether to blog more about snapback as i'm new to from scratch/greenfield coding but I guess how better to learn than to do??
  • oh yeah, i asked about an umbrella term for a bunch of stuff I'm just going to call possible technologies for now cos it feels complicated.
