Week 6: Rethinking, reinterpreting

Last week, I said I was going to try to make a much nicer data model for my Elixir / Phoenix IMDB app. I made this assumption based on the belief that the IMDB API I was using had access to the additional data I needed. Unfortunately, I realized after some research that this was not the case!

It turns out that if I want to show some of the IMDB data I’d like to display, I’d have to write a web scraper, or download, extract, and parse some of their daily-updated datasets. I’m not too interested in writing a web scraper at the moment (and I’m afraid it might break regularly when the IMDB page changes their layout), but parsing their data files seems like it could work, even if I’ll have to pay a bit to fly.io to get enough space on the server to hold the files.

But by the time I realized this (near the end of last week), I realized I’d been working on this app for a while now, and could use a break. It seems like a lot of other Recursers are thinking the same thing as we come up on the midpoint of our batch, so it’s only natural I suppose.

After some thought, I decided I wanted my next project to be working through Crafting Interpreters, since one of my fellow Recursers is going through the book currently too, and it feels like the most important gap in my knowledge that I think would be best to work through with someone else. I started the book today, and it really is such a pleasant read so far (the drawings! The examples! The tone! The lettering!) It’s a lovely introduction to an intimidating topic, and I’m really looking forward to reading it! And to sharing what I learn from reading it with you all too, of course.

I promise I’ll return later to my Elixir / Phoenix app eventually, though, to get that data read in and looking pretty! Just switching gears for a little bit.

Oh, and about that promised post about Elixir’s pattern matching: I realized that, since Elixir already has a good doc about this, my time would be be better spent doing a short teaser presentation to get people interested in the power of pattern matching instead. You can find it on Youtube here.
