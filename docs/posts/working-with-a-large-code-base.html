Working with A Large Code Base

The bigger a code base, the more tooling we need. Think about Hack at Meta, Basel or Protocol Buffers at Google, and Kafka at LinkedIn. Big tech is big on tooling. Their tooling. It helps them scale.

In Navigating Cities of Code, the author puts forward the idea that at successively larger LOC, a code base requires us to adopt more sophisticated techniques and practices. Documentation, testing, and expressive strong static types are some concrete examples 1.

The idea is great 2, but some of the techniques and practices listed (for working with code) are ambiguous, and the author’s personal experience. If we just observe the industry, I think we can all see that tooling is hugely important, and that it should be listed there too.

At one end of the scale, big tech has entire departments dedicated to (internal) tooling. A significant part of their organisation is dedicated to improving software development for all of their organisation. On the other side of the spectrum, it’s common for lots of smaller shops to have at least a platform team.

Day to day, I think we can all take inspiration from this observation to build-out more modest tooling, more frequently.
