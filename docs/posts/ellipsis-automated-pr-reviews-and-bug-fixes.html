How does Ellipsis work?

Ellipsis is an GitHub application that uses various Large Language Models to understand source code. It can also generate code.

How does Ellipsis generate code?

Ellipsis only generates code when you explicitly request it to. You can tag us in a pull request ("hey @ellipsis-dev, fix this issue...") or on an issue ("@ellipsis-dev, implement this"). We will never commit to a branch without your permission.

How does Ellipsis review code?

We'll review pull requests within a few minutes of them being opened. We'll append a summary to the PR body, so you don't have to spend time describing your changes.

Does Ellipsis store or train on my code?

No. We only read your code into memory on our secure AWS VPC when working with it. We don't store it to disk, write it to a database, or otherwise log the contents of your repository. Learn more at https://docs.ellipsis.dev/security.

Does Ellipsis commit directly to my codebase?

Never to your main branch, and only to feature branches when you've explicitly asked us to. You can configure to create new pull requests for all of its changes. Learn more at https://docs.ellipsis.dev/security.
