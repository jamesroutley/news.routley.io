Just implemented discard (discard_fragment) support in the open source AGX compiler (the Mesa driver for the Apple M1). The code's a mess but it passes the tests... One more step closer to a GPU-accelerated M1 Linux desktop üòÉ

On other GPUs, there is a single "discard if <condition>" instruction. Code like "if (x < y) { discard; }" is compiled as http://DISCARD.f32.lt x, y

AGX does not have such an instruction. In fact, it doesn't even have a discard instruction. The same is compiled as: if (x < y) { magic_instruction(0); return; } magic_instruction(1);

This magic instruction can't be DISCARD ... it runs twice. It's only DISCARD when its argument is 0. What is it?

Except for some details about helper invocations (that differ between Metal and OpenGL), discarding a fragment is equivalent to setting its sample coverage mask to 0, so it outputs no samples and hence isn't rendered.

It turns out magic_instruction writes a sample mask. So the logic is: if (x < y) { set sample mask to 0 } else { set sample mask to 1 }

Why does the compiler need to set the sample mask to 1 explicitly, when it doesn't have to in shaders that don't use discard? Th hardware has two modes controlled by a bit in the command stream: use the sample mask in the shader, or infer it.

When discard is used anywhere in the shader, the sample mask must be specified by the shader in *every* possible control flow. When discard is not used anywhere, there's no need to specify the sample mask on *any* control flow. So you'd see 0 or 2 set sample mask instructions.

Of course, the compiler can always run "set sample mask to 1" at the end of every shader, and always use the sample mask from the shader. But that's slow.
