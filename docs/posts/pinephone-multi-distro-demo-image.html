PinePhone multi-distro demo image

Here you can find a multi-boot uSD card image containing 15 different Pinephone distributions. The purpose of this image is for users to easily try current major (and some minor) Pinephone Linux distributions, without having to figure out how to flash them individually and juggle with many uSD cards.

Key points:

  • [Relatively] small 6.9GiB image containing 15 distributions:
    • Arch Linux ARM 2021-11-20
    • Arch Linux ARM / dreemurrs 20220124
    • Fedora 0.6.0
    • LuneOS 20220108
    • Maemo Leste 20220123
    • Manjaro / Phosh beta21
    • Manjaro / Plasma beta10
    • Mobian 20220116
    • pmOS / Plasma Desktop 2022-01-07
    • pmOS / GNOME 2022-01-07
    • pmOS / Plasma Mobile v21.12-20220119
    • pmOS / Phosh v21.12-20220119
    • Sailfish 20220112
    • pmOS / Sxmo v21.12-20220119
    • Ubuntu Touch 2022-01-20
    • jumpdrive 0.8
  • Runs my p-boot bootloader and my latest kernel (5.16.2 with pinephone keyboard support)
  • Based on btrfs subvolumes and file level deduplication (if you mess up, you can easily revert each distribution to original state using pre-created btrfs snapshots)
  • All included distributions work on Braveheart and later Pinephone variants (incl. 3GiB variant), thanks to sharing my kernel and bootloader
  • My modem power management driver is used for safe and reliable handling of modem power, except on distributions that use the new eg25-manager program.
  • Suspend to RAM based on the latest crust is available to all distributions (whether they actually use it or not - most don't suspend frequently enough by themselves and just turn off the display)
  • The image is made to fit a 16GiB or larger uSD cards, and also works when flashed to eMMC

Some development history and technical details can be found on my blog.

News

2022-01-26

  • All distros updated to the latest stable versions.
  • Newer kernel (5.16.2) that contains many stability fixes and power saving improvements.
  • Pinephone keyboard should be usable in all distros.
  • Supports lanscape mode p-boot build (you'll have to patch the image yourself, though). Use dd if=p-boot-landscape.bin of=multi.img bs=1024 seek=8 conv=notrunc before flashing the image (or change of= to point to an already flashed SD card block device, to update the already flashed SD card).
  • Sailfish had some trouble starting up in my quick test, and I didn't figure out why.

2021-06-02

  • All distros updated to the latest stable versions.
  • Package managers should be properly configured to safely upgrade the distributions or install new packages.
  • Newer kernel (5.13-rc4) that contains many stability fixes and power saving improvements.

2020-11-23

  • Added more distros (Manjaro, Arch dreemurrs, Jumpdrive, pmOS / plasma desktop). Removed PureOS (doesn't seem to be maintained anymore).
  • All distros updated to the latest versions.
  • p-boot has more features, auto-repeat for volume keys, scrolling menu, reboot to eMMC OS, air-gapped boot mode, battery indicator.
  • Newer kernel (5.10-rc5) with 60fps fix.

Installation

Installation is performed by decompressing image data (compressed using zstd) directly to the SD card's whole block device. For example you can use zstd -d multi.img.zst ; sudo dd if=multi.img of=/dev/sdcarddevice bs=4M oflag=direct status=progress (if you know how to identify your SD card's block device).

Verify the image was written correctly immediately after flashing by running dd if=/dev/sdcarddevice bs=1M count=10000 | sha256sum. SHA256 sum must be 39915b9d2aa2f33fd78552ac9a0e665c4aef97dd68a9f9a6c76f9fa2f0ac049e. If your OS auto-mounts new filesystems, make sure it does not do it for the SD card image, otherwise the hash may change, and this verification method will be unreliable.

You must stretch the OS partition to fit your entire uSD card (do NOT use gparted, it will just break the image silently), otherwise the free space inside the filesystem will be limited to about 0.5 GiB, which will cause distros to fail after a few boots. If you run out of space, all included distributions may become unstable. Distributions are configured to be updateable using a package manager (only kernel and u-boot updates are blocked).

Partition stretching instructions are on this page and in the README file distributed with the image.

Passwords

All passwords were changed to 1111. Root passwords are 1111. sxmo login/password is user/1111. Manjaro seems to insist on 123456.
