I discuss:

  • the structure of requests and responses
  • verbs/methods (including underused ones such as HEAD)
  • an example API that responds to POST but not GET
  • content types
  • status codes (including 451 as an Easter egg)
  • headers
  • the weird case of From:
  • the Host header and how it's used in malformed requests that try to trick a site admin into clicking spam links in server 404 logs
  • making your own headers like X-Wikimedia-Debug
  • very odd status codes and reason-phrases found in the wild
  • and ideas for further exploration
