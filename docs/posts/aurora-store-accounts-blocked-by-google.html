Description

On the latest stable release of AuroraStore and the 24/05/2023 nightly build I am unable to login with anonymous accounts.

Expected Behaviour

I expect to be able to login using an anonymous account.

Actual Behaviour

AuroraStore infintely loading and not logging in with an anonymous account.

Steps to Reproduce

  1. Install AuroraStore

  2. Open AuroraStore

  3. Click yes on every step and grant all permissions

  4. Click either Anonymous or Anonymous (insecure)

Environment

  • Device model & codename: Google Pixel 5
  • Android version: 13
  • Aurora Store version: 4.2.2
  • Installation method: Installing from F-Droid
  • OS: GrapheneOS
  • Google Play services version code: 231712044
  • Google Play Store version code: 83581220

Additional context

Today GrapheneOS pushed a 'GmsCompatConfig', 'Google Play Services', and 'Google Play Store' update.

Logging into the AuroraStore (and updating apps) worked fine yesterday.
