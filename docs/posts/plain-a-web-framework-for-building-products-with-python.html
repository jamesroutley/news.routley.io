Ecosystem

Plain is split into multiple first-party packages. Major features are optional and new ideas can evolve independently.

Foundation Packages

Building blocks for products

plain

The foundations for shipping Python code to the web — urls, views, templates, forms, and more.

plain.models

Store your data in Postgres, MySQL, or SQLite with a tried and true ORM.

plain.cache

A caching solution designed just to work with your existing database.

plain.email

App-wide email configuration and sending, with templates and attachments.

plain.sessions

Read and write to database-backed sessions.

plain.worker

An official solution for background work, backed by the database you already have.

plain.api

Build JSON APIs with the same class-based view architecture you already know.

Auth Packages

Authentication you can trust (and host yourself)

plain.auth

The foundation for adding users to your app, authenticating them to requests and restricting views.

plain.oauth

A flexible OAuth solution to support modern social logins and API access.

plain.passwords

Password-based login that works like it always has, if you want it.

plain.loginlink

Send one-time login links to users, and let them log in without a password.

plain.passkeys coming soon

Passwordless login with passkeys.

Admin Packages

The backbone of your back office

plain.admin

An admin dashboard you can fully customize, and Plain packages know how to integrate with.

plain.flags

Database-backed feature flags to control who sees what and quietly deploy progress.

plain.support

Provide better customer support without the need for third-party services.

plain.redirection

plain.pageviews
