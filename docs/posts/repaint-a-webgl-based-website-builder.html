Why another website builder?

We think there's an obvious hole in the market. It's becoming standard to design websites in a design tool, then rebuild it in a website builder. That means duplicate work, learning two tools, design-to-code translation... it's a pain. But existing website builders are miles behind the UX of Figma, and Figma can't make real websites. If it was all in one tool, it would be SO much easier. That's the concept behind Repaint. It's a design tool for real code.

How does Repaint handle responsiveness?

First you design your website for desktop. Then you make adjustments on different breakpoints. In design terms, Repaint uses autolayout and breakpoints. In code terms, we use flexbox and media queries. We've closely followed html and css. We even have responsive units. If you're unfamiliar with responsive design, our template library has lots of examples.

Can I import from Figma?

We don't support importing designs from Figma. We've thought about it. We've heard a lot of horror stories about export plugins. At best, its low on our todo list. But if you're familiar with Figma, Repaint will probably be easier than other tools, even if they have a plugin.

Is this made for beginners or professionals web designers?

Repaint is made for a range of semi-technical people: designers, developers, founders... It's definitely more technical than the average person off the street. But you don't have to be a full-time web dev.
