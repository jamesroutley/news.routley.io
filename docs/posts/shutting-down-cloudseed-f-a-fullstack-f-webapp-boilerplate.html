Shutting Down CloudSeed F# - A Fullstack F# Webapp Boilerplate

DISCLOSURE: If you buy through affiliate links, I may earn a small commission. (disclosures)

I'm shutting down CloudSeed F# - the fullstack F# webapp project boilerplate I've built and sold for the last 3 years.

Here I'll share a bit about what it was, why I'm shutting it down, and the future of CloudSeed - including how to get your hands on the code if you still want it.

What was CloudSeed F#?

CloudSeed F# was a fullstack project boilerplate. The idea was to provide a seed for any cloud-native webapp so you could get setup fast and start building your differentiating features.

It included things like:

  • DB + Migrations
  • Containerization
  • E2E testing infra with containers

And ran on some great technology:

  • DBUp - For data migrations
  • Falco - For clean, functional API scaffolding
  • Falco.Markup - For composable SSR HTML (also the inspiration for CinderblockHtml)
  • Postgres - Just use Postgres
  • And of course F#

I built it as I was learning F# to speed up my time to ship and have used it in dozens of projects over the years.

Why I'm shutting CloudSeed F# down

I'm shutting it down because I'm not personally using it much anymore and few people are buying it. I think both of these are interlinked as the less I use it, the less I write about it in my posts, and the less people end up buying it.

I still think F# is a fun language and CloudSeed F# is a great place to start an F# web app but I'm personally moving away from the F# ecosystem. As such I'm not building many projects with F# so this boilerplate isn't as useful to me and won't be getting the updates / upgrades it would through regular usage.

So at this point it's not useful enough to me to keep it running and I also don't feel comfortable selling something I'm not actively using - the bones are there but it will slowly fall out of date with best practices and library versions.
