What’s our product?

I have a vague idea of what the product should do - let’s get ChatGPT to flesh it out.

I’m creating a startup that will allow businesses to easily ask their customers to upload files. The customers should not need to have an account and the business should simply be able to share a link with the customer and then get notified when the customer has uploaded their files. Give me a high level overview of how this system would work.

Sure, here is a high-level overview of how this system could work:

  1. The business creates an account on your platform and generates a unique link that they can share with their customers.
  2. The customers receive the link and click on it to access a webpage where they can upload their files. They can also provide additional information, such as their name and contact information.
  3. The files are uploaded to a secure server, where they are stored and accessible only to the business.
  4. The business is notified when a customer uploads their files, and can view and download the files from their account on your platform.
  5. The business can also manage and organize their customer’s files, and even share them with other stakeholders or third-party applications.

Overall, the system provides a simple and secure way for businesses to collect and manage files from their customers without requiring them to create an account or install any software. Let me know if you have any other questions.

How do we build it on AWS?

Not bad - let’s see some technical details, how would we build this on AWS?

If I were to implement this solution using AWS what services should I use and how would they interact?

To implement this solution using Amazon Web Services (AWS), you could use the following services and their corresponding features:

  1. Amazon S3 (Simple Storage Service) for storing and managing the files uploaded by customers. S3 provides secure and scalable object storage, and allows you to easily access and retrieve the files using their unique keys.
  2. Amazon Cognito for managing the user accounts and authentication. With Cognito, you can easily set up a user pool and define the authentication workflow, allowing your customers to securely upload their files without the need to create an account.
  3. Amazon SNS (Simple Notification Service) for sending notifications to the business when a customer uploads their files. SNS allows you to send real-time notifications via email, SMS, or other supported channels, so the business can be alerted as soon as the files are available.
  4. Amazon Lambda for running custom backend logic and integrating with other services. With Lambda, you can write custom code to process the uploaded files, trigger notifications, or integrate with other AWS services.

The overall flow would be as follows:

  1. The business generates a unique link and shares it with their customers.
  2. The customers access the link and are redirected to a webpage hosted in Amazon S3.
  3. The customers upload their files using the S3 webpage, and provide their authentication credentials using Amazon Cognito.
  4. The files are stored in Amazon S3 and the business is notified via Amazon SNS.
  5. The business can access and manage the files from their account on your platform, using the APIs provided by Amazon S3 and Cognito.

Let me know if you have any other questions or need more information.

What if we wanted it on GCP?

  4. Google Cloud Functions for running custom backend logic and integrating with other services. With Cloud Functions, you can write custom code to process the uploaded files, trigger notifications, or integrate with other GCP services.

The overall flow would be similar to the AWS implementation, with the following differences:

  1. The customers would access the upload webpage hosted in Google Cloud Storage, and provide their authentication credentials using Firebase.
  2. The files would be stored in Google Cloud Storage and the business would be notified via Google Cloud Pub/Sub.
  3. The business could access and manage the files from their account on your platform, using the APIs provided by Cloud Storage and Firebase.

Give me the code for the lambda function that would notify users when a file is uploaded to S3

const AWS = require('aws-sdk');
const SNS = new AWS.SNS();

exports.handler = async (event) => {
    const bucket = event.Records[0].s3.bucket.name;
    const key = event.Records[0].s3.object.key;

    // Retrieve the user's email address from the file metadata
    const s3 = new AWS.S3();
    const metadata = await s3.headObject({ Bucket: bucket, Key: key }).promise();
    const email = metadata.Metadata.email;

    // Send a notification to the user via Amazon SNS
    const params = {
        Message: `Your file "${key}" has been uploaded to S3.`,
        Subject: 'File Upload Notification',
        TargetArn: `arn:aws:sns:<REGION>:<ACCOUNT>:<TOPIC>`,
        TopicArn: `arn:aws:sns:<REGION>:<ACCOUNT>:<TOPIC>`,
    };
    await SNS.publish(params).promise();

    return;
};
